Create your data

The overall purpose of a plot is to show the relationship between two or more variables.
Here is some data from: https://www.mathsisfun.com/data/scatter-xy-plots.html

In [2]:
temps = (14.2, 16.4, 11.9, 15.2, 18.5, 22.1, 19.4, 25.1, 23.4, 18.1, 22.6, 17.2) # in degrees Celsius
sales = (215, 325, 185, 332, 406, 522, 412, 614, 544, 421, 445, 408) # dollars

# For the line plot, I'll need to sort the x-axis, otherwise I'll wind up with somethign funky.
# Don't worry about how this works. Just use it if you ever need to sort your data for line plots.
temps, sales = zip(*sorted(zip(temps, sales)))

Calculate the Pearson correlation coefficient (r)

It's usually a good to include this in our plots.
It's a quick way to see the relationship between two variables in one number.
In the statistics world this is also called the Pearson product-moment coefficient.
It isn't sufficient, mind you. We still need to plot the data to get a better sense of it.
But we'll just calculate this here so we can use it later.
Here's a link that demonstrates how to calculate r by hand:
https://www.mathsisfun.com/data/correlation.html
Ain't nobody got time for that, so let's ask numpy to compute the correlation coefficient for us.

In [3]:
print(np.corrcoef(temps, sales))

[[ 1.          0.95750662]
 [ 0.95750662  1.        ]]

You see numpy.corrcoef returns a list-like structure containing 4 elements.
Two 1s (i.e., relationship of a variable with itself) and two values that actually represent the relationship between temps and sales.
That's a very strong positive correlation, by the way.
So let's set r to 0.96 and move on with our lives.
You can do that with the following. It takes the value from row 0, column 1, and rounds it two decimal places.

In [4]:
r = round(np.corrcoef(temps, sales)[0][1], 2)

Draw a scatter plot

A scatter plot is the simplest way to represent the relationship between the variables.
You can customize the style of the markers for this.
You can find a list of markers here:
http://matplotlib.org/api/markers_api.html#module-matplotlib.markers

# Draw a scatter plot
plt.scatter(temps, sales, color="blue", marker=".", linewidth=5)
plt.show()

Draw a line plot

Line plots can help us see change from one point to another
Therefore, they usually make more sense when the x-axis is "time".
But we'll draw a line plot here anyway.
I'm also including the scatter plot just so we can see the points.

plt.figure(2)

Draw multiple plots

Here's an example that plots multiple data sets.
We use color-coordination and a legend to help viewers.

In [8]:
'''
Get some new data based on what we already have.
This trick just adds small pseudo-random values (neg or pos) to the data we have.
Doing so ensures the new data doesn't look too random.
'''
temps_2 = [num + randint(-5, 5) for num in temps]
sales_2 = [num + randint(-50, 50) for num in sales]

# Sort the new data according to temps_2
temps_2, sales_2 = zip(*sorted(zip(temps_2, sales_2)))

# Get r for the new data
r_2 = round(np.corrcoef(temps_2, sales_2)[0][1], 2)

# Setup graphic. Changing my axes a bit to make sure the new data fits.
plt.figure(3)
plt.axis([5, 31, -50, 750])
plt.grid(True)
plt.title("Yearly Ice Cream Sales ($) by Temperature (\N{DEGREE SIGN}C)\nYear 1 r = {}\nYear 2 r = {}".format(r, r_2))
plt.xlabel("Temperature (\N{DEGREE SIGN}C)")
plt.ylabel("Sales ($)")

# Draw the first plot
plt.scatter(temps, sales, color="blue", marker=".", linewidth=5, zorder=1, label="Year 1")
plt.plot(temps, sales, color="red", zorder=0)

# Draw the second plot
plt.scatter(temps_2, sales_2, color="green", marker="*", linewidth=3, zorder=1, label="Year 2")
plt.plot(temps_2, sales_2, color="orange", zorder=0)

# Create the legend
plt.legend().draggable()

# Show the figure
plt.show()

Create a linear regression model

Our original data looks like it follows a linear pattern.
Let's use our algebra skills and numpy to come up with a simple model.

In [9]:
'''
Create a linear model (y = ax + b) using np.polyfit()

First argument passed represents our independent variable
Second argument passed represents our dependent variable
Third argument is the "degree" of the function, which will always be "1".
More about function degrees here: http://www.tpub.com/math1/12a.htm
'''
model = np.polyfit(temps, sales, deg=1)
print(model)

[  30.08786179 -159.47415234]

The two values returned represent a and b, respectively.
So the complete model is:
sales = 30.09 * temps - 159.47
But is this actually a good model for our data?
Let's find out by plotting it against the data itself.

Plot the model ("line of best fit") onto the data

This is an important step. It's not enough to just have the model. We have to test it.
Plotting the model onto the data helps us judge how well it fits with our natural intuition.
After all, the model is based on our assumption that the data follows a linear regression pattern.
We won't really know for certain until we see it in action. Put it to the test!

In [10]:
# Create a range of x values based on temps.
x_range = temps

# The function p sets up the y = ax + b equation for the next step
p = np.poly1d(model)    

# For every value in x_range, multiply by a and add b (i.e., just plug and chug!)
y_range = p(x_range)        

# Setup your graphic
plt.figure(4)
plt.axis([10, 26, 0, 700])
plt.grid(True)
plt.title("Ice Cream Sales ($) by Temperature (\N{DEGREE SIGN}C)\nr = {}".format(r))
plt.xlabel("Temperature (\N{DEGREE SIGN}C)")
plt.ylabel("Sales ($)")

# Draw the plot
plt.scatter(temps, sales, color="blue", marker=".", linewidth=5, zorder=0)

# Draw the regression line
plt.plot(x_range, y_range, color="red", zorder=1)

# Show the figure
plt.show()
